# Claude Code HUD 状态栏与速率限制

## 一、HUD 状态栏字段说明

Claude Code 状态栏（HUD）示例：

```
Opus 4.6 (1M context)@2.1.85 | ∠main ≡3 | ~/claude-code-learning | commit: 12h
ctx 3%/97% [1000K] in:3 out:85 | 5h:0%(4h43m) 7d:14% | ~5s
CPU:82%(6.52) Mem:13.0/16G Disk:84% up 4 days | 192.168.31.14
@ 41cc5c34-b56d-400b-ab2d-41c2638aea37
Bash:0 Skill:0 Agent:0 Edit:0
```

| 字段 | 含义 |
|------|------|
| `Opus 4.6 (1M context)@2.1.85` | 当前模型及 Claude Code 版本号 |
| `∠main ≡3` | Git 分支为 `main`；`≡3` 表示有 3 条 stash（可用 `git stash list` 验证） |
| `ctx 3%/97% [1000K]` | 上下文窗口已用 3%，剩余 97%，总量 1000K tokens |
| `in:3 out:85` | 本次会话输入 3K / 输出 85K tokens |
| `5h:0%(4h43m) 7d:14%` | 速率限制用量（见下节） |
| `~5s` | 上次响应耗时 |
| `CPU / Mem / Disk` | 系统资源占用 |
| `Bash:0 Skill:0 Agent:0 Edit:0` | 本次会话各工具调用次数 |

> **注意**：`≡3` 不是"未推送的 commit 数"，而是 stash 数量。

## 二、速率限制机制

### 两层滑动窗口

Anthropic 对 Claude Code 订阅用户实施两层速率限制：

#### 5 小时滑动窗口

- 限制**短期突发**使用，防止集中大量消耗
- "滑动"：不是固定整点重置，而是持续计算**过去 5 小时**的累计用量
- HUD 显示格式：`5h:0%(4h43m)` → 5小时窗口用了 0%，还剩 4h43m 后滑过当前高峰期

#### 7 天滑动窗口

- 限制**长期持续**使用
- 计算过去 7 天的累计消耗
- Anthropic 称此限制仅影响不到 5% 的用户

#### 两层配合示意

```
短期（5h）：防突发  ██░░░░░░░░  20%  ← 集中使用几小时后涨上来
长期（7d）：防持续  ███░░░░░░░  30%  ← 一周内累计消耗
```

任意一层达到 100% 即触发限速，需等待窗口滑过或切换低成本模型（如 Sonnet/Haiku）。

## 三、订阅计划与速率限制对照

### 个人计划

| 计划 | 价格 | 5小时窗口消息数 | 特点 |
|------|------|-----------------|------|
| **Free** | 免费 | — | 不支持 Claude Code |
| **Pro** | $20/月 | ~45 条 | 基础 Claude Code 支持 |
| **Max 5x** | $100/月 | ~225 条 | Pro 的 5 倍额度，优先访问 |
| **Max 20x** | $200/月 | ~900 条 | Pro 的 20 倍额度，最高优先级 |

### 团队计划

- **Premium**：$100/座/月（包含 Claude Code，最少 5 座）
- Standard 不支持 Claude Code

### 模型访问

所有付费计划均可访问 **Opus 4.6、Sonnet 4.6、Haiku 4.5**，主要区别在于用量额度和高峰期优先级。

## 四、Sub Agent 阻塞机制

### 默认：前台阻塞

```
主 Agent
  │
  ├─── 派出 subagent ──► 等待结果...
  │◄───────────────────────────────
  继续执行（使用 subagent 的结果）
```

适用于**后续步骤依赖 subagent 结果**的场景。

### 可选：后台非阻塞

通过 `run_in_background: true` 参数，subagent 在后台运行，主 Agent 继续执行其他工作，subagent 完成时自动通知。

### 并行多个 subagent

同一条消息内同时派出多个 subagent，并发执行，全部完成后主 Agent 再继续：

```
主 Agent
  ├─── subagent A ──┐
  ├─── subagent B ──┤ 并发执行（节省时间）
  └─── subagent C ──┘
                    │
  ◄─────────────────┘ 全部完成后继续
```

### general-purpose subagent

- 拥有**全部工具**（`*`），无限制
- 没有指定 `subagent_type` 时的**默认选择**
- 适合多轮搜索、读文件、执行命令等复杂任务
- 不确定用哪个专用 Agent 时的兜底选项
